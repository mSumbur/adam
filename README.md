2. 遵从 ESLint 或者 standardjs 等代码规范
3. 函数多多益善
4. [发布到 npm](https://github.com/muwenzi/Program-Blog/issues/12) ，发布后把链接贴到你的总结里
5. 测试文件统一归到 `test` 目录下
6. code coverage 不小于 95%
7. 使用 git hook 来自动跑用例和代码覆盖率
